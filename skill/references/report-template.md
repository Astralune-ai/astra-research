# research 报告结构与站位标准（Assemble 阶段用）

> Asher 2026-06-10 定。v1 的「结论先行 + 怀疑者一根筋 + 替客户拍板」被否。

## 站位与笔法（硬标准）
- **中立站位 + 明确观点**：不替委托方拍「投/不投、做/不做」的板。给环境、给判断、把「**若做要盯什么 / 若不做会规避或错过什么**」两面摆开，决定权留给委托方。
- **叙述化，不生硬**：像正常讲事情的文档，连贯成段。结论先行只用在每部分开头的「市场核心结论」，不主导全篇语气，更不堆 bullet 当全文。
- **新鲜判断，不闭门造车**：项目评价基于真实采集给出「它本质是什么」，不空转。
- **主章（一–六）每章配图表/插图**；收尾章（七/八/九）、执行摘要、顾问综述为结论性文字，**豁免配图**（Asher 2026-06-10 定）。
- **多轮查证（loop）**：内容不达标就倒回去继续搜，查够为止；确实搜不到的如实注明（不硬编）。具名公司/关键数据缺口优先补。
- **详尽度 / 篇幅**：报告要**自包含 + 详细**——把相关内容**全部写进正文**（具名公司带一句描述、数据带具体数字、项目方提供的资料如投资表逐项/逐表解读），**不做「摘要 + 指向 01_sources」的薄版**。行业/市场尽调目标 **≥ 2 万字**（字符级），几千行没问题，宁长勿略；不达标就补真内容，**不灌水**。
- **来源写法**：报告里的来源**一律写真实官方网址**——正文就近标官方 URL，**附录列「参考来源」按研究线汇总该线全部官方链接**。**绝不把内部 MD 文件（`01_sources/*.md`）当来源写进报告**，那只是审计轨迹、不是报告引用。
- 写作禁令同全局：禁「不是 X 而是 Y」「从 X 到 Y」等对比框架，正面直叙。

## 默认结构（行业/市场环境尽调）
0. **顾问综述 · 致委托方（目录前置，不进 TOC）** —— 站在「为客户好」的位置先给一份总结：
   - **我们的研判与定义**：项目究竟是怎么回事（新鲜定义）+ **潜在机遇** + **潜在风险**（机遇风险都摆，先机遇后风险）。
   - **关于实施的三个视角**：若决定推进 → 怎么做更稳 / 若推进不顺 → 多半卡在哪 / 若决定暂不推进 → 可基于什么考量。
   - **语气：高情商、不逼客户做选择**——三条路都摆开、每条该看清的讲透、决定权留客户；末尾点一个分水岭变量。
   - 源里用 `<!-- FRONT-OVERVIEW -->...<!-- /FRONT-OVERVIEW -->` 包住，`pdf_template.py` 自动提取、放到封面与目录之间（标题不进 TOC）。
1. **市场环境与结论**
   - (a) 结论先行：市场核心结论（中立但有观点，讲环境，不下投资指令）
   - (b) 市场饱和度与主要玩家 —— 配 **市场规模图** + **玩家格局矩阵**
   - (c) 项目评价：这是个什么项目（新鲜判断，不闭门造车）
2. **预设细分情况** —— 配 **2×2 象限表**
   - 认知度 × 市场进入 的四象限：大家都知道但没人做 / 有人做但没人知道 / 有人做但反馈不好 / 有人做且反馈好
   - **跨国差异**：同一产品在不同国家标准不同（US MIL-STD / EU·NATO STANAG / 中国 GB·军标），落在不同象限 —— 给跨市场对照表
3. **利益相关方分析（必须具名）** —— 配 **产业链图**
   - 上游 / 下游 / 横向逐环讲谁关键、谁有议价权、卡点在哪
   - **硬要求：点名具体公司，不许"一些厂商"。** 若目标市场含中国本土 → 列全本土相关公司；若涉国外准入 → 上游、下游**各至少 3 个**具名全球对手。
4. **业务逻辑分析（怎么做这件事）**
   - (a) **目标客户群**：目标客户是谁（按段细分 + 具名潜在买家 + 难度排序）
   - (b) **资源与执行**：手里要什么"牌"、怎么操作、是否要加设备、人工成本（对照行业基准）
   - (c) **资金管理**：资本结构、月烧/跑道、单位经济、资金管理方案（配 **投资结构图**）
5. **客观视角对比** —— 配 **对比表**
   - (a) 对方怎么描述 vs 它实际属于什么 ; (b) 客观看它到底是什么 ; (c) 竞品/相关方怎么做的
6. **两面摆开**：若投要盯什么 / 若不投会规避或错过什么 + **分水岭变量**（不替委托方拍板）
7. **调查覆盖与局限**：说明查了哪几条线、置信度、哪些是查不到的缺口（多为项目方内部数据，注明需向项目方索取，不硬编）
8. **待核实硬缺口**

> 这是行业/市场环境尽调的默认骨架；其他研究类型（通用深调/公司尽调/社媒）按需裁剪，但「中立 + 明确观点、叙述化、两面摆开、每段配图」的站位标准通用。

## 配图怎么做
- **主章（一–六）每章至少一张图/表（收尾章与摘要豁免）。图必须逻辑清晰**：实线为主、单向流（上游→制造→下游）；**忌折叠交叉的虚线**——虚线越多越乱 = 逻辑没理清（Asher 明确反对）。画之前先把逻辑列清楚再画。
- **结构图**（产业链/流程）→ mermaid `flowchart`；**数据图**（市场规模/价格）→ mermaid `xychart-beta`。渲染：`~/.local/bin/mermaid <f>.mmd -o assets/<f>.svg`，`![标题](assets/<f>.svg)` 嵌进报告。
- **象限 / 2×2** → mermaid `quadrantChart` 需要 mmdc（chromium，常未装 → 渲染不了），**回退用 2×2 markdown 表格**（更稳、到处能渲染）。
- **保留 Mermaid 源**：每张图都留 `assets/<f>.mmd`，正文内嵌渲好的 `assets/<f>.svg`。
- **转 Word / PDF**：导出时用 `~/.local/bin/mermaid <f>.mmd -o assets/<f>.svg`（beautiful-mermaid 引擎）把 Mermaid 重渲为 SVG 再嵌入，确保图随文档走。
- 客户档要更精致的图 → `astra-graphic` / `astra-imagen`。

## 客户交付：导出高级 PDF（脚本化管线，2026-06-10 v0.3 起）
**一条命令**：`bash ~/.claude/commands/research/_repo/scripts/build.sh <工作区>` —— 依次跑 ① `render_charts.sh`（`assets/*.mmd` → beautiful-mermaid SVG → 删 Google Fonts @import → headless Chrome 截 2× PNG）② `verify_report.py`（七项 lint，**不过不出 PDF**）③ `build_report.py`（weasyprint 渲 PDF：封面信纸 / 顾问综述前置 / 目录真页码 / 底部页码）。依赖钉在 `~/.venvs/research`（首次 `bash _repo/scripts/setup.sh` 一键装；本机另需 `~/.local/bin/mermaid` + Chrome + PingFang·Songti 字体）。
- **报告 meta 写在 REPORT.md 顶部 YAML front-matter**（`title / kicker / subtitle / date / classification / report_type / out / brand / footnote`），构建脚本读 meta 渲封面——**复用零改码**，不再复制脚本手改。
- **信纸 = brand pack**：`_repo/brand/<名>/`（`cover.png` 封面整页背景 + `inner-soft.png` 内页整页背景），front-matter `brand:` 选用，默认 `astralune`；客户机部署加自己的 brand pack 即可。Astralune 信纸正式来源在公司品牌柜 `~/Company/X.H._Noctuer_Holdings_Pty_Ltd/07_品牌资产_branding/Astralune/素材_assets/信纸_letterhead/`。
- **配色 = 黑白灰单色（禁蓝禁金）**：炭黑 #26282c 标题/表头 + 银灰 #9fa1a6 点缀 + 近黑正文 + 浅灰斑马；表头炭黑底白字。
- **核心原则（Asher 定）**：真信纸当整页背景，正文顺着留白区流，**不另加角标/页眉/logo 占信纸的位置**；文字压在淡水印上是正常的。内页边距 `40mm 24mm 20mm 24mm`，表格/引用框背景半透明让水印透出；封面标题 Songti 25pt。
- 技术坑（**已全部被脚本吸收，无需手操**，列出仅为排障）：① Mermaid SVG 含 CSS 变量，librsvg/weasyprint 渲成黑块 → Chrome 截 PNG；② `@page{background}` 默认画在边距内被挤压 → `background-size:210mm 297mm; background-position:-左margin -上margin` 全幅定位；③ CommonMark 列表前要空行、连续 `> **字段**` 会并段 → `_fix_lists`/`_fix_bq` 自动修（内联 ①②③ 枚举仍需源里加 `<br>`）；④ TOC 真页码 = python-markdown `toc`(slugify_unicode) + CSS `target-counter(attr(href url), page)`。
- 首跑实例：`~/Asher/reports/PrintLive2026_3D打印无气轮胎_尽调_2026-06-09/REPORT.md`（v3 现行；v1 怀疑者版 / v2 四段版归档在 `_archive/`，对照看进化）。
